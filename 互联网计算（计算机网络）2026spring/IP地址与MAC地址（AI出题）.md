### 📝 题型一：网络层子网划分与 CIDR 计算题

**【题目】**

观察“无线局域网适配器 WLAN”的配置信息，该主机的 IPv4 地址为 `10.54.230.65`，子网掩码为 `255.255.224.0`。请完成以下计算：

1. 请用 **CIDR（无分类域间路由）** 符号表示该主机的 IP 地址与网络前缀。
2. 计算该主机所在的**子网网络地址（Network ID）**。
3. 计算该子网的**广播地址（Broadcast Address）**。
4. 计算该子网内可以分配给主机的**有效 IP 地址范围**，以及该子网**最多能容纳的主机数量**。

**【详细解析与答案】**

1. **CIDR 表示法**：

   - 子网掩码 `255.255.224.0` 转换为二进制：前两个字节共 16 位全为 1；第三个字节 `224` 转换为二进制是 `11100000`（前 3 位为 1）；第四个字节全为 0。
   - 因此，网络前缀长度 = $8 + 8 + 3 = 19$ 位。
   - **答案**：`10.54.230.65/19`。

2. **计算子网网络地址**：

   - 将 IP 地址的第三个字节 `230`（二进制：`11100110`）与子网掩码的第三个字节 `224`（二进制：`11100000`）进行按位与（AND）操作：

     `11100110 & 11100000 = 11100000`（十进制为 `224`）。

   - 剩余的主机位（第四字节）全部置 0。

   - **答案**：子网网络地址为 `10.54.224.0/19`。

3. **计算子网广播地址**：

   - 保持前 19 位网络前缀不变，将后 $32 - 19 = 13$ 位主机位全部置为 1。
   - 第三个字节变为 `11111111`（即 `224 | 31 = 255`），第四个字节变为 `255`。
   - **答案**：广播地址为 `10.54.255.255`。

4. **有效 IP 范围与主机数量**：

   - 有效的主机 IP 地址范围是从网络地址的下一个地址开始，到广播地址的前一个地址结束。
   - **有效 IP 范围**：`10.54.224.1` ~ `10.54.255.254`。
   - 主机位共有 13 位，减去网络地址和广播地址 2 个特殊地址。
   - **最大容纳主机数**：$2^{13} - 2 = 8192 - 2 = 8190$ 台。

### 📝 题型二：网络层与链路层协同——跨网段路由与 ARP 简答题

**【题目】**

假设该主机想要向互联网上的某台公共 Web 服务器（IP 地址为 `182.61.200.7`）发送一个 IP 数据报。请结合日志信息与协议原理回答下列问题：

1. 主机是如何判断目标服务器是位于“本网段（局域网内）”还是“外网段（远程网络）”的？
2. 该 IP 数据报在被网卡发送出去时，其 IP 首部中的**源 IP 地址**和**目的 IP 地址**分别是什么？该数据报在本地局域网内需要交付给哪一个具体的 IP 地址？
3. 主机在数据链路层封装 MAC 帧时，需要知道上述本地交付节点的 MAC 地址。请问主机使用什么协议来获取该地址？如果在本地缓存中没有找到该地址，主机会发送什么类型的帧（单播还是广播）？为什么？

**【详细解析与答案】**

1. **网络位置判断机制**：
   - 主机使用自己的子网掩码 `255.255.224.0` 与目的 IP `182.61.200.7` 进行**按位与**运算，得到结果 `182.61.192.0`。
   - 主机将其与自身的子网地址 `10.54.224.0` 进行对比。因为两者不相等，所以判定目标服务器位于**外网段（远程网络）**。
2. **IP 寻址与下一跳**：
   - IP 首部属于端到端网络层控制，其**源 IP** 为 `10.54.230.65`，**目的 IP** 始终为 `182.61.200.7`。
   - 既然是外网段，数据报必须在本地先交付给**默认网关（Default Gateway）**，对应的 IP 地址为 `10.54.224.1`。
3. **链路层地址解析（ARP 协议）**：
   - 主机使用 ARP（地址解析协议）来获取网关的物理地址。
   - 如果本地 ARP 缓存未命中，主机会发送一个 **ARP 请求帧**，该帧在链路层是**广播帧**（目的 MAC 地址为 `FF-FF-FF-FF-FF-FF`）。
   - **原因**：因为主机此时并不知道网关的 MAC 地址，无法进行单播通信；通过广播形式，局域网内的所有节点都能收到该请求，而网关 `10.54.224.1` 发现请求的是自己的 IP 后，会以单播形式向主机返回自己的 MAC 地址。

### 📝 题型三：数据链路层——MAC 地址与层级架构简答题

**【题目】**

1. 观察日志中 WLAN 适配器的物理地址 `18-93-41-53-94-4F`。请问标准的以太网/Wi-Fi MAC 地址是多少位（多少字节）？结合课件中关于 MAC 地址和 IP 地址的类比，简述它们在网络体系结构中作用和作用范围的区别。
2. 日志中显示了多个适配器（如以太网 2、以太网 3、WLAN、蓝牙等），它们各自拥有完全不同的“物理地址”。请问：一台计算机主机可以同时拥有多个不同的 MAC 地址吗？为什么？

**【详细解析与答案】**

1. **MAC 与 IP 地址的 scope（范围）差异**：
   - 标准 MAC 地址为 **48 位（6 字节）**。
   - **区别**（参考课件第一部分 Slide 4 & 6）：
     - **MAC 地址**：是物理层/链路层地址，具有全球唯一性和便携性。它的作用是“本地的”，仅用于在同一个子网（局域网）内将帧从一个物理接口交付到另一个直接相连的物理接口。**类比：个人的身份证号（SSN）**。
     - **IP 地址**：是网络层地址，用于在**全球互联网**范围内进行三层转发和路由选择。IP 地址是不具有便携性的，其网络前缀取决于设备当前所接入的子网位置。**类比：邮寄信件时的邮政地址（Postal Address）**。
2. **多 MAC 地址的合理性**：
   - **可以**。因为 MAC 地址并不是分配给“整台计算机主机”的，而是分配给网络接口（NIC，即网卡）的。
   - 每一个物理网卡（如 Intel Wi-Fi 网卡、Realtek 有线网卡）或虚拟网卡（如 VPN 虚拟接口），只要它需要独立接入某个链路层广播域，就必须在硬件 ROM 中烧录或由软件分配一个唯一的 48 位 MAC 地址。

### 📝 题型四：综合网络应用——DHCP 租约与跨子网服务分析题

**【题目】**

1. 日志中显示：`获得租约的时间: 2026年6月30日 9:18:45`，且 `DHCP 服务器 : 10.54.224.1`。请简述该主机在接入 Wi-Fi 时，通过哪四个核心步骤（消息交互）从该 DHCP 服务器处动态获取到当前 IP 配置的？
2. 观察日志中的 DNS 服务器地址为 `10.60.1.2` 和 `10.60.1.3`。请通过数学计算说明：这两台 DNS 服务器是否与当前主机处于**同一个本地子网内**？主机在向该 DNS 服务器发起域名解析请求时，其发出的 MAC 帧的目的硬件地址应该填什么？

**【详细解析与答案】**

1. **DHCP 四步交互时序**：
   - **DHCP Discover（发现）**：客户端通过广播发送发现报文，寻找局域网内的 DHCP 服务器。
   - **DHCP Offer（提供）**：DHCP 服务器 `10.54.224.1` 收到后，向客户端返回包含预分配 IP（`10.54.230.65` 等配置）的提供报文。
   - **DHCP Request（请求）**：客户端选择该配置，向全网广播请求报文，通知所有服务器自己接受了该分配。
   - **DHCP ACK（确认）**：服务器 `10.54.224.1` 发送确认报文，正式将该 IP 租借给客户端，租约开始生效。
2. **DNS 服务器位置与路由计算**：
   - **计算判断**：主机的子网掩码为 `255.255.224.0`，已知本地子网网络地址为 `10.54.224.0`（有效范围是 `10.54.224.1` ~ `10.54.255.254`）。
   - 将 DNS 服务器的 IP `10.60.1.2` 与子网掩码作逻辑与，得到其所在网络为 `10.60.0.0`。
   - **结论**：因为 `10.60.0.0 != 10.54.224.0`（或者 `10.60.1.2` 不在 `10.54.224.1 ~ 10.54.255.254` 范围内），所以 **DNS 服务器不属于本地子网**，它位于学校或企业网络内的另一个远程网段。
   - **链路层封装选择**：由于目标是外网段，主机在网络层将目的 IP 填为 `10.60.1.2`，但在链路层封装 MAC 帧时，**目的硬件地址（MAC 地址）必须填写默认网关 `10.54.224.1` 的 MAC 地址**，由网关路由器帮其完成后续的跨网段路由转发。
